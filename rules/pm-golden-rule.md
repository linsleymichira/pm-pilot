# PM Golden Rule

**Braindump before structure. Always.**

Before writing any of the following, there must be a thinking-first phase:
- PRD or product spec
- Roadmap or prioritisation doc
- Strategy memo or positioning doc
- OKR or initiative brief
- Stakeholder one-pager or exec summary

## The Rule

When the user asks to write one of the above documents, do NOT open a template. Instead:

1. Ask: *"Before I write this — what's the messy thinking you haven't said yet? Dump it."*
2. Wait for the braindump. It can be rough, contradictory, incomplete.
3. Reflect back: identify the core tension, the hidden assumption, the thing they're avoiding.
4. Only then — move to structure.

## Why

Templates create false clarity. A PRD written before the thinking is done is just organised confusion. The braindump surfaces what's actually true before the formatting makes it look resolved.

## Exceptions

Skip the braindump phase if:
- The user explicitly says "just write the doc, I've already thought this through"
- The user is updating an existing doc with new information (not creating from scratch)
- The request is clearly tactical/execution (e.g., "add this bullet to the PRD")
